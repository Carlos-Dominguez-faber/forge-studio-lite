# Estructuras narrativas para carruseles (`/carrusel`)

Catálogo de referencia para clase en vivo. Adaptado de las 35 narrativas del repo
[`santmun/historias-ig-skill`](https://github.com/santmun/historias-ig-skill), reescritas
para el formato carrusel de Instagram (slides cuadrados 1080×1080) que produce el comando
`/carrusel`.

---

## 1. Qué es una estructura narrativa (y por qué importa)

Una **estructura narrativa** es el esqueleto de tu carrusel: el orden en que las slides
sueltan la información para que la persona deslice hasta el final. No es decoración, es
ingeniería de atención. La primera slide gana el deslizamiento; cada slide intermedia tiene
que dejar una pregunta abierta que solo la siguiente cierra; la última cobra con un CTA.

El carrusel **no vende con el diseño, vende con la historia**. Un carrusel feo con una
estructura fuerte retiene más que uno bonito sin tensión. El diseño solo evita fricción
(legibilidad, jerarquía, marca); la estructura es la que hace que la persona llegue a la
slide donde le pides la acción. Por eso eliges la estructura *antes* de escribir una sola
slide, y por eso este documento existe.

**Cómo eliges una estructura según tu objetivo:**

| Objetivo | Familia de estructuras |
|---|---|
| **Lead magnet** (capturar contacto / DM) | Storytelling tipo chisme, Soltamos la bomba, Microentrenamiento, Demostración → CTA directa |
| **Urgencia** (que actúen ya) | Predicción / alerta, El error fatal, Esto te puede estar frenando |
| **Humanización** (que confíen en ti) | Diario personal, Storytelling de error propio, Lección inesperada, Narrativa de rutina |
| **Educativo** (autoridad / valor) | Explicación de concepto, Lo que nadie te cuenta, Desmentida estratégica, Detección de creencia |
| **Prueba social** (validar con resultados) | Transformación del cliente, Mini documental de caso, Evolución visible |
| **Libre / creativo** (cuando no encaja nada) | Apertura cinematográfica, Rompe patrón visual, Atmósfera / escena sensitiva |

Regla práctica: si dudas, **Tier 1**. Las narrativas tipo chisme y bomba enganchan a casi
cualquier audiencia porque explotan curiosidad pura. Baja de tier solo cuando el contenido
es inherentemente informativo y la audiencia ya te busca por valor.

## 2. Catálogo de estructuras (por tier de enganche)

El tier ordena por **potencial de retención**: Tier 1 son las que más detienen el scroll
(curiosidad/tensión tipo chisme), bajando hacia las educativas/informativas, que enganchan
menos en frío pero construyen autoridad. La columna "slides" es el conteo recomendado;
ajusta ±1 según tu contenido. La primera slide siempre es el **hook**, la última siempre el
**CTA**.

## 3. Cómo se mapea a `/carrusel`

El comando `/carrusel` no genera imágenes con IA: cada slide es una **página HTML
autocontenida** que Playwright captura como PNG a 1080×1080. La estructura narrativa que
elegiste arriba se materializa en archivos:

```
./carousels/<slug>/
├── STORYBOARD.md                 # 1 sección por slide (rol + texto)
├── slides/source/
│   ├── slide-01.html             # HOOK  ← siempre la primera slide de la estructura
│   ├── slide-02.html             # punto / problema / prueba
│   ├── slide-03.html             # ...
│   └── slide-NN.html             # CTA   ← siempre la última
└── slides/
    └── slide-NN.png              # output del screenshot
```

Reglas de mapeo:

- **Una slide del esqueleto = un `slide-NN.html`.** El número del archivo es el orden de
  deslizamiento. `slide-01.html` es el hook; el último archivo es el CTA.
- **HTML autocontenido.** Cada slide lleva sus tokens de marca copiados *inline* en el
  `<style>` (no `<link>` a `brand.css` externo: cada slide es autónoma y debe funcionar
  desde cualquier ruta). La fuente se carga vía CDN.
- **Roles de slide** (úsalos en el `STORYBOARD.md` para no perder la estructura):
  - `hook` → slide-01. Detiene el scroll. Frase corta, contraste alto, una sola idea.
  - `punto` → desarrollo. Cada slide avanza el argumento y deja pregunta abierta.
  - `prueba` → datos, ejemplo, antes/después, captura. Sostiene la credibilidad.
  - `CTA` → última slide. Una acción clara ("comenta X", "guarda esto", "ve al link").
- **Keyword-highlighting.** Las palabras clave de cada slide se resaltan con el color
  primario de la marca usando `var(--primary)`. Es el patrón visual que ancla la mirada:
  el ojo salta de keyword en keyword mientras desliza. Resalta 1–3 palabras por slide, nunca
  frases enteras (pierde el efecto).

Flujo de trabajo: eliges estructura → escribes `STORYBOARD.md` mapeando cada beat del
esqueleto a una slide con su rol → escribes los `slide-NN.html` → corres el pipeline TS que
hace el screenshot batch. El agente decide la estructura; el HTML es el contrato.

## 4. Plantilla de slide

Esqueleto mínimo de una slide de carrusel. 1080×1080, tokens de marca en `:root`, contenido
centrado con flex, y el patrón de keyword-highlighting con `var(--primary)`.

```html
<!DOCTYPE html>
<html lang="es">
<head>
<meta charset="UTF-8" />
<style>
  /* Fuente vía CDN (Geist como ejemplo; sustituye por la de tu marca) */
  @font-face {
    font-family: 'Geist';
    src: url('https://cdn.jsdelivr.net/npm/geist@1.5.1/dist/fonts/geist-sans/Geist-Variable.woff2') format('woff2');
    font-weight: 100 900;
  }

  /* Tokens de marca copiados inline desde brand.css (NO link externo) */
  :root {
    --bg:      #0B0F1A;   /* fondo */
    --text:    #FFFFFF;   /* texto principal */
    --primary: #2563EB;   /* highlight de keywords */
    --muted:   #94A3B8;   /* texto secundario */
    --font-display: 'Geist', -apple-system, sans-serif;
  }

  * { margin: 0; padding: 0; box-sizing: border-box; }

  .slide {
    width: 1080px;
    height: 1080px;
    background: var(--bg);
    color: var(--text);
    font-family: var(--font-display);
    display: flex;
    flex-direction: column;
    justify-content: center;   /* centrado vertical */
    align-items: flex-start;
    padding: 96px;
    gap: 32px;
  }

  .kicker { font-size: 28px; color: var(--muted); letter-spacing: 2px; text-transform: uppercase; }
  .headline { font-size: 92px; font-weight: 800; line-height: 1.05; }

  /* keyword-highlighting: resalta 1–3 palabras con el color primario */
  .hl { color: var(--primary); }
</style>
</head>
<body>
  <div class="slide">
    <div class="kicker">01 · Hook</div>
    <div class="headline">
      Nadie te dijo que tu carrusel
      <span class="hl">no vende con el diseño</span>.
    </div>
  </div>
</body>
</html>
```

Para el CTA (última slide), reusa la misma `.slide` pero cambia el `.kicker` a `CTA`, pon la
acción en el `.headline` y resalta el verbo o la palabra de acción con `.hl`. Mantén el mismo
sistema visual en todas las slides: solo cambia el contenido, no el esqueleto.
